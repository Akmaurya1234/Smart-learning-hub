Introduction

Java is a high-level, object-oriented programming language known for its portability and performance.

  • Write Once, Run Anywhere: Platform-independent via JVM
  • Robust: Strong memory management and exception handling
  • Widely Used: For web, mobile, and enterprise development

Syntax

Java syntax defines the structure of Java programs and the rules for writing code.

  • Class: Defined using class keyword
  • Method: Code logic goes inside methods like main()
  • Semicolon: Ends statements
  • Braces: Used for blocks of code

Data Types

Data types define the type of data a variable can hold in Java.

  • Primitive Types: int, float, double, char, boolean
  • Reference Types: Arrays, Classes, Interfaces, etc.
  • Type Casting: Used to convert between types

Variables

Variables store data values that can be used and modified during program execution.

  • Declaration: type variable = value;
  • Scope: local, instance, or static
  • Initialization: Variables must be initialized before use

Operators

Operators are used to perform operations on variables and values.

  • Arithmetic: +, -, *, /, %
  • Relational: ==, !=, >, <, >=, <=
  • Logical: &&, ||, !

Control Statements

Control statements allow you to control the flow of execution.

  • if / else: Executes code conditionally
  • switch: Selects among multiple options
  • Nested Conditions: Multiple conditions inside each other

Loops

Loops are used to execute a block of code repeatedly.

  • for: Loop with initialization, condition, and increment
  • while: Executes as long as condition is true
  • do-while: Executes at least once

Functions

Functions (methods) contain blocks of code that perform specific tasks.

  • Declaration: returnType name(params)
  • Static/Instance: Can be static or object-based
  • Return: May return a value using return

Arrays

Arrays store multiple values of the same type in a single variable.

  • Declaration: type[] arrayName = new type[size];
  • Initialization: int[] a = {1, 2, 3};
  • Access: array[index]

Pointers

Java does not support explicit pointer manipulation for safety and security reasons.

  • References: Java uses references instead of pointers
  • No pointer arithmetic: Not allowed in Java
  • Memory Management: Handled by JVM (Garbage Collector)

Structures

Java does not have a struct keyword like C; it uses classes instead.

  • Class: Java's class is more powerful than C-style struct
  • Object-Oriented: Encapsulation and methods included
  • Reusable: Classes can be extended and reused

File Handling

Java provides file handling through classes in the java.io and java.nio packages.

  • Read: Scanner, FileReader, BufferedReader
  • Write: FileWriter, PrintWriter
  • Operations: create, read, write, delete

OOP Concepts

Java is a fully object-oriented language built on 4 main principles:

  • Encapsulation
  • Abstraction
  • Inheritance
  • Polymorphism

Inheritance

Inheritance is the process where one class acquires properties of another.

  • extends: Used to inherit from another class
  • Super class: The parent class
  • Sub class: The derived class

Polymorphism

Polymorphism allows objects to be treated as instances of their parent class.

  • Compile-time: Method overloading
  • Run-time: Method overriding
  • Dynamic Binding: Decides method to execute at runtime

Encapsulation

Encapsulation is the process of wrapping data and code together as a single unit.

  • Private fields: Hide data from outside access
  • Public methods: Control how data is accessed or changed
  • Data Security: Ensures data integrity

Abstraction

Abstraction hides internal details and shows only the functionality to the user.

  • Abstract class: Contains abstract methods
  • Interface: Defines methods without implementation
  • Implemented: By subclasses or classes
