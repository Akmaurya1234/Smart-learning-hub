JavaScript

JavaScript is a scripting language used to make web pages interactive. It works alongside HTML and CSS. It's the backbone of dynamic functionality on websites, allowing you to control behavior, interactivity, and data.

  • Stands for JavaScript.
  • Client-side scripting language (runs in browser).
  • Can also run on server using Node.js.
  • Makes web pages interactive (sliders, forms, popups, etc).
  • Used in both frontend and backend development.
  • Supported by all modern browsers.
  • Works with HTML and CSS to build full websites.

JavaScript Features:

  • Lightweight, interpreted language.
  • Object-Oriented Programming support.
  • Event-driven programming (responds to user actions).
  • Supports functional programming style too.
  • Dynamic typing — no need to declare data types.
  • Platform independent (runs on any browser).
  • Used to build web apps, mobile apps, games, servers, etc.
  • Integrates well with other languages/APIs.

  • WebStorm

What is a scripting language?

A scripting language is used to automate tasks that could alternatively be executed one by one by a human operator. JavaScript is a scripting language that runs inside the browser and allows you to add dynamic behavior to your website. It does not need compilation and runs directly in the browser.

JavaScript Syntax

  • JavaScript syntax is the set of rules that define a correctly structured JavaScript program.
  • It includes how to declare variables using var, let, and const.
  • JavaScript supports different data types like strings, numbers, booleans, arrays, and objects.
  • Statements are the building blocks and usually end with a semicolon (;).
  • JavaScript is case-sensitive: MyVar and myvar are different.
  • Comments can be added using // for single-line or /* */ for multi-line.
  • Functions are defined using the function keyword and allow code reuse.
  • JavaScript runs in browsers and can also be used on servers with Node.js.
  • Code can be placed inside the <script> tag in HTML or in separate .js files.
  • Understanding syntax is key to writing clean, error-free JavaScript code.

JavaScript Statements

  • JavaScript programs are built using individual commands called statements.
  • A statement might declare a variable, perform a calculation, or call a function.
  • Each statement ends with a semicolon `;` (optional, but recommended).
  • Statements are executed top to bottom, in the order they appear.
  • You can write multiple statements on separate lines for readability.
  • JavaScript automatically adds semicolons in some cases, but it's not always reliable.
  • Statements can include loops, conditions, assignments, or function calls.
  • Grouping statements together in functions helps organize your code.
  • Misplacing a statement or semicolon might break your code or cause unexpected results.
  • In the example, each line performs a step: declare, calculate, and display a result.

Semicolons

  • Semicolons (`;`) in JavaScript mark the end of a statement.
  • While they are optional in many cases, using them prevents ambiguous code.
  • JavaScript has a feature called Automatic Semicolon Insertion.
  • This means the interpreter adds semicolons where it thinks they belong.
  • However, this guess can fail and result in logic or syntax errors.
  • Using semicolons manually ensures your code behaves as expected.
  • Every line in the example below ends with a semicolon for clarity and safety.
  • Some minifiers and build tools may require semicolons to function properly.
  • It’s considered good coding style and a best practice to always use them.
  • Remember, clear code avoids bugs — semicolons make code easier to read too.

Comments

  • Comments are used to explain code and are ignored by the JavaScript engine.
  • They help make the code easier to read and understand later.
  • Single-line comments begin with // and go to the end of the line.
  • Multi-line comments start with /* and end with */.
  • They can span multiple lines and are good for longer notes or instructions.
  • Use comments to describe what blocks of code are doing or why a line exists.
  • Comments are also useful when debugging or testing your code.
  • You can temporarily disable a line by turning it into a comment.
  • Good commenting improves collaboration, especially in team projects.
  • Remember not to over-comment obvious code — balance is key.

Identifiers

  • Identifiers are the names used to identify variables, functions, arrays, objects, etc.
  • They must begin with a letter (a-z or A-Z), underscore (_) or dollar sign ($).
  • Identifiers cannot begin with a digit (0-9).
  • JavaScript identifiers are case-sensitive; value and Value are different.
  • You can use letters, digits, underscores, and dollar signs in identifiers after the first character.
  • Using meaningful identifiers improves code readability and maintenance.
  • Reserved keywords like var, function, etc., cannot be used as identifiers.
  • A good identifier name should describe the purpose of the variable or function.
  • Avoid very short names like x unless in small loops or temporary values.
  • Consistent naming conventions (like camelCase) are recommended in JavaScript.

Case Sensitivity

  • JavaScript is case-sensitive, which means Name and name are considered different.
  • This rule applies to all identifiers like variables, functions, and object properties.
  • If you declare let total = 100; and later write Total, it will be undefined.
  • Inconsistent casing often causes bugs, especially in large applications.
  • Always stick to a consistent naming convention like camelCase in JavaScript.
  • Case sensitivity also applies to built-in objects and methods like Math vs math.
  • When using DOM elements, getElementById must be spelled exactly with proper case.
  • Errors due to casing are common in beginners, so reviewing names carefully is crucial.
  • Editors with autocomplete features help avoid these casing issues.
  • Using a linter tool can also help catch mismatched cases in variable names.

Whitespace & Line Breaks

  • Whitespace includes spaces, tabs, and line breaks in your code.
  • JavaScript usually ignores whitespace, so it's used mostly for readability.
  • Code with good spacing is easier to read and maintain.
  • You can break code into multiple lines for better visual structure.
  • Example: let x = 5; and let x=5; work the same.
  • Extra spaces inside variable names are not allowed (e.g., let my var is invalid).
  • Consistent indentation improves collaboration in teams.
  • Whitespace does matter inside strings and regular expressions.
  • Tools like Prettier can auto-format whitespace for you.
  • Clean spacing helps prevent logical errors during debugging.

Literals

  • Literals are fixed values written directly into the JavaScript code.
  • Common literals include numbers (e.g., 10), strings (e.g., "Hello"), booleans (e.g., true), etc.
  • String literals are enclosed in single or double quotes.
  • Numeric literals include integers and floating-point numbers.
  • Boolean literals are either true or false.
  • Array literals are enclosed in square brackets like [1, 2, 3].
  • Object literals use curly braces with key-value pairs.
  • Null and undefined are also considered literals in JavaScript.
  • Literals are used in assignments and function arguments frequently.
  • They help form the structure of expressions in JavaScript.

Keywords

  • Keywords are reserved words in JavaScript with special meanings.
  • You cannot use them as variable or function names.
  • Examples include let, const, var, if, else, etc.
  • They are the building blocks of the JavaScript language.
  • let and const are used for declaring variables.
  • if and else are used for decision-making.
  • function is used to declare a function.
  • Using a keyword incorrectly results in a syntax error.
  • Some old keywords like var are still supported for backward compatibility.
  • Always check for updated reserved words in modern JavaScript versions.

Expressions

  • Expressions are combinations of values, variables, and operators that evaluate to a single result.
  • They form the building blocks of JavaScript logic and computation.
  • An example is 5 + 2, which returns 7.
  • Expressions can be arithmetic (a * b), string ('Hello' + 'World'), or logical (a && b).
  • They can also include function calls like Math.max(5, 10).
  • JavaScript evaluates expressions from left to right using operator precedence.
  • You can assign the result of an expression to a variable: let total = price + tax;.
  • Expressions are used everywhere: in conditionals, loops, returns, and DOM operations.
  • Some expressions don’t use operators directly, like 'Hi' or true.
  • Understanding expressions is key to writing functional JavaScript code.

Code Blocks

  • Code blocks are groups of one or more statements enclosed in curly braces { }.
  • They are used to define the scope of execution in structures like if, for, and while.
  • Each block runs only when its condition or logic is met.
  • A block can contain variable declarations, function calls, or other control structures.
  • JavaScript treats code inside a block as a single unit.
  • Variables declared inside a block are only accessible within that block (block scope).
  • Blocks help organize logic and improve code readability.
  • You can also use nested code blocks inside other blocks or functions.
  • Functions themselves are also a type of code block.
  • Using code blocks is essential for controlling the flow and structure of your code.

Variables and Data Types

  • Variables in JavaScript are used to store data values that can be used and changed later in the program.
  • You can declare variables using var (old), let (modern), or const (constant).
  • let and const are block-scoped and safer to use in modern code.
  • Data types define the type of value a variable holds, such as number, string, or boolean.
  • Common primitive data types in JavaScript include: Number, String, Boolean, Undefined, and Null.
  • JavaScript also supports complex types like Object and Array for storing multiple values.
  • Example of a string: let name = "John";, a number: let age = 25;.
  • Use the typeof operator to check the data type of a variable (e.g., typeof name).
  • Variables must be named using letters, $, or _, and cannot start with numbers.
  • Understanding variables and data types helps you build flexible and error-free programs.

Declaring Variables

Use var, let, or const to declare variables.

  • var is function scoped and was used in older JavaScript versions.
  • let is block scoped and recommended for modern usage when values may change.
  • const is also block scoped and used for values that should not change.
  • Variables declared with let or const are not hoisted like var.
  • Variable names must begin with a letter, $, or _.
  • JavaScript variables are case-sensitive: name and Name are different.
  • You can store different types of values: strings, numbers, booleans, arrays, and objects.
  • let is ideal when the variable's value will be updated later.
  • const is ideal for fixed values like configurations.
  • Using let and const improves code safety and readability.

Variables in JavaScript

JavaScript variables are used to store data that can be reused or changed in a program.

  • var – Function-scoped, used in older versions. Example: var age = 30;
  • let – Block-scoped, allows reassignment. Example: let name = "John";
  • const – Block-scoped, used for constant values. Example: const pi = 3.14;
  • Variables can store different types of values like strings, numbers, booleans, arrays, and objects.
  • Variable names must start with a letter, $, or _ and are case-sensitive.

var

var is the oldest way to declare a variable in JavaScript.

  • var is function-scoped, not block-scoped.
  • It can be redeclared and updated in the same scope.
  • Hoisted to the top with an initial value of undefined.
  • Not recommended for modern JavaScript due to scope issues.
  • Example: var name = "John";

let

let is used to declare variables that can be changed later.

  • let is block-scoped and doesn’t allow redeclaration in the same scope.
  • Best for variables whose values will change.
  • Hoisted but not initialized (gives ReferenceError if accessed before declaration).
  • Improves code readability and reduces bugs.
  • Example: let count = 5;

const

const is used to declare variables whose value should never change.

  • const is block-scoped like let.
  • Must be assigned a value during declaration.
  • Once assigned, its value cannot be changed or reassigned.
  • Used for constants like configuration or fixed values.
  • Example: const appName = "MyApp";

Data Types in JavaScript

JavaScript supports numbers, strings, booleans, objects, arrays, etc.

  • JavaScript is a dynamically typed language, which means variables can hold any data type.
  • Number includes both integers and floating-point values (e.g., 5, 3.14).
  • String represents text and is enclosed in quotes (e.g., "Hello" or 'World').
  • Boolean has only two values: true or false, used for logical decisions.
  • Object is a collection of key-value pairs (e.g., {name: "John", age: 30}).
  • Array is an ordered list of values (e.g., [1, 2, 3]), which can be mixed types.
  • Undefined means a variable has been declared but has not yet been assigned a value.
  • Null represents an intentionally empty value or "no value."
  • Use the typeof operator to check the type of a variable at runtime.
  • Understanding data types is essential for writing accurate and bug-free JavaScript code.

Number

Number is used for any numeric value, including decimals and negatives.

  • Can be integers or floating-point numbers.
  • Example: let price = 99.99;
  • Arithmetic operations can be performed: +, -, *, /
  • typeof price gives "number"
  • Infinity and NaN (Not a Number) are special number values.

String

String represents text and is written in quotes (single or double).

  • Used to store words, sentences, or characters.
  • Example: let city = "Delhi";
  • Use typeof to check: typeof city gives "string".
  • Can include escape characters like \n for newline or \t for tab.
  • Concatenate strings using +: "Hello " + "World"

Boolean

Boolean represents only two values: true or false.

  • Used in conditional logic and comparisons.
  • Example: let isOnline = false;
  • 5 > 3 returns true
  • typeof isOnline gives "boolean"
  • Very useful in if-else statements and loops.

Object

Object stores data in key-value pairs using curly braces.

  • Example: let car = { brand: "Toyota", year: 2023 };
  • Access with dot or bracket notation: car.brand or car["brand"]
  • Used to structure related data together.
  • typeof car gives "object"
  • Nested objects and arrays are also allowed.

Array

Array stores multiple values in a single variable using square brackets.

  • Example: let fruits = ["apple", "banana", "cherry"];
  • Access elements by index: fruits[0] gives "apple"
  • Arrays are a type of object, so typeof fruits gives "object"
  • Can store mixed data types: [1, "two", true]
  • Used for loops, lists, and bulk data management.

Undefined

When a variable is declared but no value is assigned, it's undefined.

  • Example: let score;
  • typeof score gives "undefined"
  • Commonly seen in functions with no return value.
  • Helps check if a value has been set or not.
  • Avoid using it as a manual value.

Null

null is an intentional absence of any object value.

  • Used to reset or clear variables manually.
  • Example: let result = null;
  • typeof null returns "object" (this is a known JS quirk)
  • Different from undefined, which means "not assigned"
  • Helps represent empty or unknown data explicitly.

JavaScript Operators

JavaScript operators are symbols used to perform operations on variables and values. They are essential in every JavaScript program, and understanding them helps you write better and more logical code. There are various types of operators, including arithmetic, comparison, assignment, logical, bitwise, string, typeof, and ternary.

Arithmetic Operators

Used to perform mathematical operations like addition, subtraction, etc.

  • Examples: +, -, *, /, %
  • These operators work with numeric values.
  • + adds two numbers; * multiplies them.
  • They return a numeric result based on the operation.
  • Example: let sum = 5 + 3;

Assignment Operators

Used to assign values to variables.

  • Main operator is = (assignment).
  • Compound assignment: +=, -=, *= etc.
  • x += 5 is same as x = x + 5.
  • Useful for quick updates to variables.
  • Example: let x = 10; x *= 2;

Comparison Operators

Used to compare two values and return a boolean result.

  • Examples: ==, ===, !=, >, <, >=
  • == checks value only, === checks value and type.
  • Returns true or false.
  • Used often in conditions and loops.
  • Example: 5 < 10 returns true.

Logical Operators

Used to combine multiple conditions (booleans).

  • && is AND, || is OR, ! is NOT.
  • && returns true only if both conditions are true.
  • || returns true if at least one condition is true.
  • ! reverses the boolean value.
  • Example: true && false returns false.

Conditional (Ternary) Operator

A shorthand way of writing if-else condition in a single line.

  • Syntax: condition ? trueResult : falseResult
  • Returns one of two values depending on the condition.
  • Useful for quick decisions inside expressions.
  • Example: (x > 10) ? "Big" : "Small"
  • Helps reduce multiple lines of code.

Modulus Operator

Used to find the remainder of a division operation.

  • Syntax: a % b
  • Returns the remainder when a is divided by b.
  • Example: 10 % 3 returns 1 (10 divided by 3 leaves remainder 1).
  • Used to check for even or odd numbers or cycle through an array.
  • Example: 5 % 2 returns 1.

Increment and Decrement Operators

Used to increase or decrease the value of a variable.

  • ++ is the increment operator (increases the value by 1).
  • -- is the decrement operator (decreases the value by 1).
  • Can be used in both prefix and postfix forms.
  • Example: a++ increases the value of a by 1.
  • Example: a-- decreases the value of a by 1.

Equality Operators

Used to compare values and return a boolean result.

  • == (loose equality): compares values, ignores type.
  • === (strict equality): compares both value and type.
  • != (loose inequality): checks if values are not equal.
  • !== (strict inequality): checks if both value and type are different.
  • Example: 5 == '5' returns true, but 5 === '5' returns false.

Delete Operator

Used to delete properties from objects.

  • Syntax: delete object.property
  • Removes a property from an object.
  • Returns true if the property is successfully deleted.
  • Does not affect the object's prototype chain.
  • Example: delete obj.age removes the "age" property.

Unary Operators

Used to perform operations on a single operand.

  • + can also be used as a unary operator to convert a string to a number.
  • - can be used to negate a number.
  • ++ increases the operand by 1.
  • -- decreases the operand by 1.
  • Example: + '10' converts the string to number 10.

Exponentiation Operator

Used to raise the left operand to the power of the right operand.

  • Syntax: a ** b
  • Returns a raised to the power of b.
  • Example: 2 ** 3 returns 8 (2 raised to the power 3).
  • This operator is similar to the Math.pow() method.

Division Operator

Used to divide the left operand by the right operand.

  • Syntax: a / b
  • Returns the result of a divided by b.
  • Example: 10 / 3 returns 3.3333 (result of division).
  • If division by zero occurs, the result will be Infinity or -Infinity.

Multiplication Operator

Used to multiply two operands.

  • Syntax: a * b
  • Returns the result of multiplying a and b.
  • Example: 10 * 5 returns 50.
  • Used for mathematical operations involving multiplication.

Subtraction Operator

Used to subtract the right operand from the left operand.

  • Syntax: a - b
  • Returns the result of subtracting b from a.
  • Example: 20 - 5 returns 15.
  • Used to calculate differences between numbers.

Concatenation Operator

Used to join two strings together.

  • Syntax: a + b
  • Returns the concatenated result of two strings.
  • Example: 'Hello' + ' ' + 'World' returns 'Hello World'.
  • This operator is primarily used for string operations.

String Properties and Methods

JavaScript provides various operators and methods to manipulate strings.

  • .length property returns the number of characters in a string.
  • .charAt() method returns the character at a specified index.
  • .toUpperCase() converts a string to uppercase.
  • .toLowerCase() converts a string to lowercase.
  • Example: 'hello'.toUpperCase() returns HELLO.

Conditional Statements

Conditional statements allow you to execute code based on whether a condition is true or false.

  • if statement: Checks if a condition is true, and if it is, the block of code inside the if statement is executed.
  • else statement: Executes code if the if condition is false.
  • Syntax: if (condition) { // execute code } else { // execute code }
  • Example: If the condition age >= 18 is true, it will display 'You are an adult.'. Otherwise, it will display 'You are a minor.'.
  • Comparison operators: You can use comparison operators like ==, !=, >, <, etc., to check conditions.
  • Conditional statements are essential for making decisions in your code based on dynamic inputs.

if Statement

Executes a block of code if a specified condition is true.

  • if statement: It checks whether a condition is true and, if it is, runs the block of code inside the if statement.
  • Example: if (age >= 18) checks if the age is greater than or equal to 18. If true, it will display 'Adult'.
  • Syntax: if (condition) { // code to run if true }
  • This statement is often used when you want to check if something is true and take action accordingly.

2. if-else Statement

Executes a block of code if a specified condition is true, otherwise executes an alternative block of code.

  • if statement: It checks if a condition is true and, if it is, executes the block of code inside the if block.
  • else statement: If the condition in the if statement is false, it executes the block of code inside the else block.
  • Example: if (age >= 18) checks if age is greater than or equal to 18. If true, it displays 'Adult', otherwise it displays 'Minor'.
  • Syntax: if (condition) { // code to run if true } else { // code to run if false }
  • Used when you need two possibilities: one for true and one for false.

3. if-else if-else Statement

Used when you have multiple conditions to check. Executes the block of code corresponding to the first true condition.

  • if statement: Checks the first condition, and if true, runs the associated block of code.
  • else if statement: If the first condition is false, it checks the next condition.
  • else statement: If all previous conditions are false, it runs the block of code inside the else block.
  • Example: if (age >= 18) checks if age is greater than or equal to 18. If not, it checks if the age is between 13 and 18, and if neither is true, it considers the person a child.
  • Syntax: if (condition) { // code } else if (condition) { // code } else { // code }
  • This statement is used when there are more than two conditions to evaluate.

4. Switch Statement

Used to execute one of many code blocks based on different conditions.

  • switch statement: Evaluates an expression once and matches the result with each case.
  • case statement: Each case represents a possible value of the expression being evaluated. The corresponding block is executed if it matches.
  • default statement: This block executes if none of the case values match.
  • Example: switch(day) checks the value of day and prints the corresponding day name.
  • Syntax: switch (expression) { case value: // code break; ... default: // code }
  • This is useful when checking multiple values of the same variable.

Functions in JavaScript

Functions are reusable blocks of code that can take inputs (called parameters), perform a task, and optionally return a result. Functions help to avoid code repetition and make programs more modular and easier to maintain.

  • function declaration: You define a function using the function keyword, followed by the function name, parentheses, and curly braces for the code block.
  • Parameters: These are values passed into the function, allowing you to make the function more flexible and reusable with different inputs.
  • Return Value: A function can return a result to the calling code using the return statement.
  • Syntax: function functionName(parameters) { // code }
  • Function Invocation: To use the function, you invoke it by calling its name and passing values for the parameters.

Parts of a Function

  • Function Declaration: This is where the function is defined with a name, parameters (optional), and the task it should perform. Example: function greet(name) { return 'Hello, ' + name; }
  • Parameters: These are placeholders inside the parentheses where values are passed into the function when it is called. For example, in function greet(name), name is a parameter.
  • Return Value: A function can send a result back to the caller using the return keyword. For example, return 'Hello, ' + name returns the greeting message to the caller.
  • Function Invocation: To execute the function, you invoke it by calling its name and providing the necessary arguments for the parameters. Example: greet('John') calls the function with 'John' as the argument.

Regular Function

A regular function is declared using the function keyword. It can accept parameters and return a value.

  • Syntax: function functionName(parameters) { // code }
  • Usage: Regular functions are used when you need reusable code.
  • Example: function greet() { return 'Hello, World!'; }

Anonymous Function

An anonymous function is a function without a name. It is often assigned to a variable or passed as an argument.

  • Syntax: let functionName = function() { // code }
  • Usage: Used for short, temporary tasks like event handlers or passing as arguments.
  • Example: let greet = function() { return 'Hello, World!'; }

Arrow Function

Arrow functions provide a more concise way to write functions, especially for simple tasks and callbacks.

  • Syntax: const functionName = () => { // code }
  • Usage: Often used for short functions or callbacks.
  • Example: const greet = () => { return 'Hello, World!'; }

Immediately Invoked Function Expression (IIFE)

An IIFE is a function that is executed immediately after it is defined. It helps avoid polluting the global namespace.

  • Syntax: (function() { // code })();
  • Usage: Used to run code once and create private scopes.
  • Example: (function() { console.log('I am an IIFE'); })();

Constructor Function

Constructor functions are used to create new objects. They serve as a blueprint for creating instances with similar properties.

  • Syntax: function Person(name, age) { // code }
  • Usage: Used to create objects with similar structures.
  • Example: function Person(name, age) { this.name = name; }

Generator Function

Generator functions allow you to pause and resume execution, yielding multiple values over time.

  • Syntax: function* functionName() { // code }
  • Usage: Used for handling asynchronous code or iterating over large data sets.
  • Example: function* greet() { yield 'Hello'; }

Recursive Function

A recursive function calls itself in order to break down a complex task into simpler tasks.

  • Syntax: function functionName() { // code }
  • Usage: Used when a problem can be broken down into smaller, similar sub-problems.
  • Example: function factorial(n) { if (n === 0) return 1; }

Callback Function

Callback functions are functions that are passed as arguments to other functions and are executed after the completion of the main function.

  • Syntax: function functionName(callback) { // code }
  • Usage: Commonly used for handling asynchronous tasks or event-driven programming.
  • Example: function greet(name, callback) { callback(); }

Function with Arguments and Return Value

This function accepts arguments and returns a value after performing an operation.

  • Arguments: Values passed into the function when it is called.
  • Return Value: The result of the function, sent back to the caller.
  • Example: The function add(a, b) takes two arguments and returns their sum.
  • Syntax: function functionName(arguments) { return value; }

Function with Arguments but No Return Value

This function accepts arguments but does not return a value.

  • Arguments: Values passed into the function when it is called.
  • No Return Value: The function performs an action but doesn't return a value.
  • Example: The function greet(name) takes a name as an argument and logs a greeting message.
  • Syntax: function functionName(arguments) { // perform action }

Function without Arguments but with Return Value

This function does not accept arguments but returns a value when called.

  • No Arguments: The function does not require any input.
  • Return Value: The function returns a value to the caller.
  • Example: The function getPi() returns the value of pi (3.14).
  • Syntax: function functionName() { return value; }

Function without Arguments and No Return Value

This function does not accept arguments and does not return any value.

  • No Arguments: The function does not require any input.
  • No Return Value: The function performs an action but does not return a value.
  • Example: The function sayHello() logs "Hello, World!" to the console.
  • Syntax: function functionName() { // perform action }

Scope

Scope defines the context in which variables are accessible. It determines where a variable can be accessed in the code.

  • Global Scope: Variables declared outside any function can be accessed from anywhere in the code.
  • Local Scope: Variables declared inside a function can only be accessed within that function.
  • Block Scope: Variables declared with let and const inside a block (e.g., inside an if or for loop) are only accessible within that block.
  • Example: The variable globalVar is accessible inside the function testScope, but localVar is not accessible outside the function.

Closure

A closure is created when an inner function retains access to the variables of its outer function, even after the outer function has finished executing.

  • Definition: Closures allow inner functions to "remember" the scope in which they were created, even if the outer function has already completed.
  • Example: The inner function inner() can access the variable outerVar, even after outer() has finished executing.
  • Usage: Closures are used for creating private variables or maintaining state between function calls.

Objects and Prototypes

In JavaScript, objects are collections of key-value pairs, where each key (or property) is associated with a value (could be any data type).

  • Objects: Objects are used to store collections of data and more complex entities.
  • Example: let person = { name: 'John', age: 30, greet: function() { document.write('Hello, ' + this.name); }};
  • Prototypes: Every JavaScript object has a prototype, which is another object. Prototypes allow objects to inherit methods and properties from other objects.
  • Example: A prototype can be used to share properties or methods across instances of objects, reducing redundancy.
  • This is useful for creating more efficient and maintainable code by reusing common functionality across multiple objects.

Arrays and Methods

JavaScript arrays store multiple values in a single variable. Arrays are ordered collections, where each value can be accessed using an index starting from 0.

  • Arrays: Arrays allow storing of different data types, including strings, numbers, and objects.
  • Methods: Methods are built-in functions that allow you to manipulate or retrieve information from an array.
  • Example: fruits.join(', ') converts the array into a string with each item separated by a comma.
  • Arrays are essential for storing and organizing data, and methods like push(), pop(), and shift() are commonly used to modify arrays.
  • This is just one example of how methods interact with arrays to return a new representation or modify its contents.

Arrays

An array is a special type of object used to store multiple values in a single variable. Arrays are ordered collections of data.

  • Syntax: let arrayName = [value1, value2, value3];
  • Accessing Elements: You can access individual elements using their index, starting from 0.
  • Example: let fruits = ['apple', 'banana', 'cherry']; document.write(fruits[0]); displays apple, and document.write(fruits[1]); displays banana.
  • Methods: Arrays have built-in methods like push(), pop(), shift(), unshift(), concat(), and slice().

push() Method

  • push() adds one or more elements to the end of an array.
  • Syntax: array.push(element1, element2, ...);
  • It returns the new length of the array.
  • Example: fruits.push('cherry') adds 'cherry' to the end.
  • Commonly used to add items to an array dynamically.

pop() Method

  • pop() removes the last element from an array.
  • Syntax: array.pop();
  • The method returns the removed element.
  • Example: fruits.pop() removes 'cherry' from the array.
  • Used to remove items from the end of an array.

shift() Method

  • shift() removes the first element from an array.
  • Syntax: array.shift();
  • The method returns the removed element.
  • Example: fruits.shift() removes 'apple' from the array.
  • Used to remove the first item of an array.

unshift() Method

  • unshift() adds one or more elements to the beginning of an array.
  • Syntax: array.unshift(element1, element2, ...);
  • The method returns the new length of the array.
  • Example: fruits.unshift('apple') adds 'apple' to the start.
  • Commonly used to prepend items to an array.

concat() Method

  • concat() combines two or more arrays into a new array.
  • Syntax: array1.concat(array2, array3, ...);
  • The method returns a new array without modifying the original arrays.
  • Example: fruits.concat(moreFruits) merges two arrays.
  • Used to merge multiple arrays into one array.

For Loop

Executes a block of code a specified number of times.

  • for loop: This loop is used when you know in advance how many times you want to execute a statement or a block of code.
  • Syntax: for(initialization; condition; increment/decrement)
  • Example: The loop for(let i = 0; i < 5; i++) will run the code inside the loop 5 times (from 0 to 4).
  • This loop is particularly useful when you want to iterate over an array or perform a task a certain number of times.

While Loop

Executes a block of code as long as a specified condition is true.

  • while loop: This loop runs the code block as long as the condition evaluates to true.
  • Syntax: while(condition) { // code }
  • Example: The loop while(i < 5) will run as long as i is less than 5.
  • Unlike the for loop, you don’t define the iteration inside the loop itself. You need to manually update the loop counter (e.g., i++).
  • Useful when you don’t know exactly how many times you need to iterate, but you have a condition to check.

Do-While Loop

Executes a block of code once before checking the condition, and then repeats the loop as long as the condition is true.

  • do-while loop: This loop is similar to the while loop, but the block of code is executed at least once before the condition is tested.
  • Syntax: do { // code } while(condition);
  • Example: The loop will first execute the code block and then check the condition. If i is less than 5, it will continue.
  • Useful when you want the code to run at least once, regardless of whether the condition is true.

For...in Loop

Used to loop through the properties of an object.

  • for...in loop: This loop is used for iterating over the properties of an object.
  • Syntax: for (let key in object) { // code }
  • Example: The loop will iterate over the properties of the person object and print each key and its corresponding value.
  • This loop is useful when you need to access both the property names and their values in an object.

getElementById()

getElementById() is used to select an element by its unique ID attribute.

  • Returns: The element with the specified ID or null if no element is found.
  • Usage: You can use this method to access elements and then modify their properties or content.
  • Example: let element = document.getElementById('myElement'); selects the element with the ID of 'myElement'.

getElementsByClassName()

getElementsByClassName() is used to select all elements with a specified class name.

  • Returns: A live HTMLCollection of elements with the given class name.
  • Usage: It is used when you want to select multiple elements with the same class.
  • Example: let elements = document.getElementsByClassName('myClass'); selects all elements with the class `myClass`.

querySelector()

querySelector() returns the first element that matches a specified CSS selector.

  • Returns: The first matching element or null if no match is found.
  • Usage: You can pass any valid CSS selector to target an element.
  • Example: let element = document.querySelector('.myClass'); selects the first element with the class `myClass`.

querySelectorAll()

querySelectorAll() returns a **NodeList** of all elements that match the given CSS selector.

  • Returns: A static NodeList containing all matching elements.
  • Usage: Use this method when you need to select multiple elements that match a CSS selector.
  • Example: let elements = document.querySelectorAll('.myClass'); selects all elements with the class `myClass`.

let and const

let and const are used to declare variables in modern JavaScript.

  • let allows you to declare a block-scoped variable that can be reassigned.
  • const creates a block-scoped variable that cannot be reassigned after being initialized.
  • Example: let x = 10;, const y = 20;

Arrow Functions

Arrow functions provide a more concise syntax for writing functions.

  • They are anonymous functions and do not have their own this.
  • Example: const sum = (a, b) => a + b;

Template Literals

Template literals provide a way to embed expressions inside strings.

  • They are wrapped in backticks `` instead of quotes.
  • Variables or expressions can be embedded using ${}.
  • Example: let greeting = `Hello, ${name}!`;

Destructuring Assignment

Destructuring allows you to unpack values from arrays or properties from objects into variables.

  • For objects: const {name, age} = person;
  • For arrays: const [first, second] = array;

Spread and Rest Operators

The ... operator has two purposes: spreading and collecting values.

  • Spread: Used to spread elements of an array or object. Example: const arr2 = [...arr1, 4, 5];
  • Rest: Used to collect remaining elements into a single variable. Example: const [first, ...rest] = arr;

Default Parameters

Function parameters can have default values if no argument is passed.

  • Example: function greet(name = 'Guest') { ... }
  • If no value is provided for name, it will default to 'Guest'.

Classes

Classes are a blueprint for creating objects in JavaScript. They were introduced in ES6.

  • class allows you to define a template for objects.
  • Example: class Person { constructor(name, age) { this.name = name; this.age = age; } }

Modules

Modules allow you to split your code into reusable components.

  • import allows you to load functions, objects, or variables from other files.
  • export makes code available to other files.
  • Example: import { greet } from './greet.js';
